## 2. ขอบเขตฟีเจอร์และโมดูล (Feature Scope)

### โมดูลที่ 1: ระบบสั่งอาหารและแจ้งเตือนพนักงาน (Food Ordering & Staff Alert)
- ลูกค้าสั่งอาหารผ่านเว็บไซด์โดยระบุเลขโต๊ะ (สแกน QR Code ประจำโต๊ะ)
- ระบบส่งการแจ้งเตือน (Notification) เข้ากลุ่ม LINE พนักงานทันทีผ่าน **LINE Notify API**
- มีหน้า Admin Dashboard เพื่อให้พนักงานกดยืนยันออเดอร์และเปลี่ยนสถานะ (กำลังทำ -> เสิร์ฟแล้ว)

### โมดูลที่ 2: ระบบชำระเงิน (Payment System)
- **ตัวเลือกที่ 1:** จ่ายออนไลน์ทันที โดยระบบจะเจนรูปภาพ **QR Code พร้อมเพย์ (PromptPay)** ตามยอดสุทธิ ให้ลูกค้าสแกนจ่าย แล้วให้พนักงานกดยืนยันรับเงินที่หน้าเคาน์เตอร์ (Manual Verify เพื่อประหยัดค่าธรรมเนียม)
- **ตัวเลือกที่ 2:** เลือกไปชำระเงินสด/บัตรเครดิตที่เคาท์เตอร์ตอนเช็คเอาท์

### โมดูลที่ 3: ระบบสอนเกม และ Auto GM (Game Guide & AI GM)
- **ระบบสอนเกม:** หน้าค้นหากฎเกมและวิธีเล่นแบบสรุปสั้น เข้าใจง่าย อ่านจบใน 3 นาที และมีคลิปสอนซึ่งเป็นลิงค์จาก youtube เราจะสร้างและอัพโหลดเอาไว้
- **Auto GM:** ระบบช่วยดำเนินเกมอัตโนมัติ (เช่น จับเวลา, สุ่มบทบาท, เปิดเพลงบิ้วอารมณ์)
- **Dice Shop Werewolf:** มินิเกมและระบบช่วย GM สำหรับเกม Werewolf โดยเฉพาะ (ช่วยสุ่มบทบาท แจกการ์ดผ่านหน้าจอมือถือของแต่ละคน และมีบอทพูดดำเนินเรื่องในแต่ละคืน)

### โมดูลที่ 4: ระบบ Ranking ของเกม Werewolf และ สมาชิก (Member & Leaderboard)
- เก็บแต้มจากการเล่นเกม มินิเกมบนเว็บ หรือยอดใช้จ่าย
- หน้า Leaderboard แสดงอันดับผู้เล่นสูงสุดประจำสัปดาห์/เดือน เพื่อสร้าง Community

### โมดูลที่ 5: Web Mini-Games
- มินิเกมง่ายๆ บนเว็บไซต์ให้ลูกค้าเล่นฆ่าเวลาระหว่างรออาหาร หรือรอเพื่อน หรือให้คนที่มาใช้งานเว็บได้เล่นได้ (เช่น เกมทายใจ, เกมสุ่มดวง, เกมกดแข่งความเร็ว)

## 3. สถาปัตยกรรมทางเทคนิค (Technical Stack)
- **Framework:** Next.js (App Router, Version ล่าสุด)
- **Language:** TypeScript (Strict Mode)
- **Database:** SQLite (สำหรับเก็บข้อมูลในเครื่อง โครงสร้างแบบเบาและเร็ว)
- **Styling:** Tailwind CSS + Shadcn UI (เน้น Clean & Responsive)
- **State Management:** React Context หรือ Zustand (หลีกเลี่ยงความซับซ้อนของ Redux)
- **Infrastructure:** พัฒนาบน MacBook Pro M1 Pro และวางแผน Deploy บน Cloud แผนฟรี (เช่น Render หรือ Fly.io)

## 4. กฎเหล็กในการเขียนโค้ดสำหรับ Claude (Coding Rules)

1. **Token-Saving Strategy:** ห้ามเขียนโค้ด Boilerplate ที่ยาวเกินไป ให้เขียนเฉพาะสิ่งที่จำเป็นและแยกโมดูลชัดเจน (Modular Design)
2. **Component Isolation:** แยก Component ให้ชัดเจน เช่น `components/orders/`, `components/werewolf/` เพื่อให้ง่ายต่อการเรียกใช้และประหยัด Context ในการอ่านโค้ด
3. **No Unused Code:** ห้ามทิ้งโค้ดที่ไม่ได้ใช้หรือคอมเมนต์ขยะไว้ในโปรเจกต์
4. **Mobile-First CSS:** ทุกหน้าเว็บต้องดูดีที่สุดบนสมาร์ตโฟน (iPhone/Android) ก่อนหน้าจอ Desktop
5. **Simplicity First:** ฟังก์ชันไหนที่ทำแบบธรรมดาแล้วตอบโจทย์ (เช่น QR PromptPay แบบพนักงานกดยืนยันเอง) ไม่ต้องพยายามเชื่อมต่อระบบ API จ่ายเงินภายนอกที่ซับซ้อน เว้นแต่จะสั่งเจาะจง

## 5. แผนการพัฒนาที่แนะนำ (Development Roadmap Context)
- **Phase 1:** สร้างฐานข้อมูล SQLite + ระบบสั่งอาหารพื้นฐาน + LINE Notify (หัวใจหลักของร้าน)
- **Phase 2:** หน้าจอพนักงาน (Admin Dashboard) + ระบบชำระเงินแบบกึ่งอัตโนมัติ
- **Phase 3:** ระบบบอร์ดเกม (Werewolf Companion & Auto GM)
- **Phase 4:** ระบบ Ranking สมาชิก และ Mini-games เสริมบนเว็บ
