# Order Lifecycle & Close Conditions

## สถานะออเดอร์ (Order Status)

```
PENDING → CONFIRMED → PAID → SERVED
                   ↘ CANCELLED
```

## เงื่อนไขการปิดออเดอร์ (Close Conditions)

ออเดอร์จะปิด (เปลี่ยนเป็น SERVED) ได้ก็ต่อเมื่อ **ครบทุกเงื่อนไข**:

| เงื่อนไข | วิธีตรวจสอบ |
|---|---|
| ✅ ชำระเงินครบแล้ว | `order.status === "PAID"` |
| ✅ อาหารพร้อมและส่งถึงโต๊ะแล้ว | ครัวกด "เสร็จแล้ว" (ทุก item มี `kitchenServedAt`) + พนักงานกด "เสิร์ฟแล้ว" |
| ✅ พิมพ์ใบเสร็จแล้ว | กด "🖨️ พิมพ์ใบเสร็จ · ปิดออเดอร์" |

ปุ่ม "ปิดออเดอร์" จะปรากฏก็ต่อเมื่อทั้ง 3 เงื่อนไขด้านบนพร้อม

---

## ขั้นตอนรายละเอียด

### 1. PENDING — รอรับออเดอร์

- ลูกค้าสั่งอาหาร ระบบสร้างออเดอร์ใหม่
- แจ้งเตือนพนักงานด้วยเสียงและ notification
- พนักงานต้องเลือกดำเนินการตามวิธีชำระ:

| วิธีชำระ | การดำเนินการ |
|---|---|
| **CASH** | กด "รับชำระแล้ว" → ข้ามไป PAID โดยตรง |
| **PROMPTPAY** | รอลูกค้าส่งสลิป → ยืนยันสลิป → PAID |
| **TAB** (จ่ายตอนเช็กเอาท์) | พนักงานกด "รับออเดอร์" → CONFIRMED, ส่งครัวทำ |

### 2. CONFIRMED — รับออเดอร์แล้ว รอชำระ

- ครัวทำอาหาร (แต่ละ item มี `kitchenServedAt` เมื่อเสร็จ)
- สำหรับ TAB orders: รอลูกค้าพร้อมจ่าย
- เสียงแจ้งเตือนดังซ้ำขณะยังไม่ได้รับออเดอร์

### 3. ครัวเสร็จ (kitchenDone)

- ทุก item มี `kitchenServedAt` → ถือว่าอาหารพร้อม
- ระบบส่งเสียงแจ้งเตือน "อาหารพร้อม" ซ้ำจนกว่าพนักงานจะกด **"เสิร์ฟแล้ว"**
- ปุ่ม **"🍽️ เสิร์ฟแล้ว — หยุดแจ้งเตือน"** ปรากฏบนการ์ดออเดอร์ทุกใบที่ครัวเสร็จแล้ว
- การกดเสิร์ฟ: บันทึกใน localStorage เท่านั้น — **ไม่ได้ปิดบิล**

### 4. PAID — ชำระเงินแล้ว

- สำหรับ TAB: เมื่อลูกค้าพร้อมจ่าย พนักงานเลือกวิธีชำระ (เงินสด/สแกน)
- เมื่อชำระสำเร็จ → สถานะเปลี่ยนเป็น PAID
- ถ้าครัวยังทำอยู่: แสดง "รอครัวยืนยันว่าเสร็จ"
- ถ้าครัวเสร็จแล้วแต่ยังไม่เสิร์ฟ: แสดง "กดเสิร์ฟก่อน"

### 5. SERVED — ปิดออเดอร์ (ขั้นตอนสุดท้าย)

- ปุ่ม **"🖨️ พิมพ์ใบเสร็จ · ปิดออเดอร์"** ปรากฏเมื่อครบเงื่อนไข
- กดปุ่มนี้จะ: พิมพ์ใบเสร็จ + เปลี่ยนสถานะเป็น SERVED + ออเดอร์หายออกจาก queue

---

## วิธีชำระเงิน (Payment Methods)

| Method | ความหมาย |
|---|---|
| `CASH` | จ่ายเงินสดที่เคาน์เตอร์ |
| `PROMPTPAY` | โอนเงิน/QR สแกน |
| `TAB` | ชำระตอนเช็กเอาท์ (รวมกับบิลตี้) |
| `UNSET` | ยังไม่เลือกวิธีชำระ |

## เสียงแจ้งเตือน

| เหตุการณ์ | เสียง | หยุดเมื่อ |
|---|---|---|
| ออเดอร์ใหม่เข้า | Alert Sound | พนักงานรับออเดอร์ (สถานะเปลี่ยน) |
| อาหารพร้อมจากครัว | Kitchen Sound | พนักงานกด "เสิร์ฟแล้ว" |
